Build a City Dataset & Practice Clustering

Module Overview

We are interested in understanding neighborhood change. These data-driven approaches to the phenomenon use machine-learning algorithms to “discover” coherent communities within the city by grouping census tracts into groups that minimize within-group differences and maximize between-group differences.

In Lab 3 you applied clustering techniques to Phoenix data provided for you. In this module you will build your own city dataset from scratch using Census data and the tidycensus API, construct a Dorling cartogram, and then apply the same clustering methods to your chosen city. This lab is the foundation for your final project.

Learning Objectives

After completing this module, you will be able to:

  1. Use the CBSA-to-FIPS crosswalk to identify counties in a Metropolitan Statistical Area (MSA).
  2. Download census tract shapefiles and population data using the tidycensus API.
  3. Merge spatial data with harmonized Longitudinal Tract Database (LTDB) census variables.
  4. Transform a shapefile into a Dorling cartogram that corrects for tract size bias.
  5. Apply Gaussian mixture model clustering (mclust) to identify neighborhood typologies.
  6. Compare clustering results across different variable sets and composite indices.
  7. Assign interpretive labels to clusters based on percentile profile charts.

Saving Your Cartogram for Future Use

You will re-use your cartogram in future labs and in the final project. You can create your cartogram from scratch each time, or save it as a GeoJSON file and re-load it when needed.

Save your cartogram (sf pipeline):

library( sf )

# project to standard lat-lon coordinate system
my_dorling_sf <- st_transform( my_dorling_sf, crs = 4326 )

# write to file
st_write( my_dorling_sf, "my_dorling.geojson", delete_dsn = TRUE )

Load your cartogram:

library( sf )

# load from local file path
my_dorling_sf <- st_read( "my_dorling.geojson" )

plot( st_geometry( my_dorling_sf ) )

Important Technical Notes:

  • Do NOT load library(maptools) — it has been retired from CRAN and is no longer available.
  • Use st_transform() (from the sf package) instead of spTransform() (from the retired rgdal package).
  • Use st_crs(3395) instead of CRS("+init=epsg:3395").
  • Use st_drop_geometry() instead of @data to extract data from spatial objects.
