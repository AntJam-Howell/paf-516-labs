Interactive Dashboard (Final Project)

Module Overview and Objectives

In this capstone module, you will transform your Lab 6 static policy brief into an interactive Quarto Dashboard. The dashboard uses format: dashboard and relies on crosstalk for linked interactivity between maps and tables — no Shiny server required. The result is a self-contained HTML file published to RPubs so anyone with the URL can explore your analysis.

This project synthesizes everything you have learned across Labs 1–6: index construction, choropleth mapping, spatial joins, LISA hot spot analysis, temporal change analysis, and policy communication. The new skills are dashboard layout (tabs, columns, value boxes) and crosstalk interactivity (clicking a polygon on a leaflet map highlights the corresponding row in a DT table, and vice versa).

After completing this module, you will be able to:

  • Build a Quarto Dashboard with multiple tabs and value boxes
  • Use crosstalk::SharedData to link interactive maps and tables
  • Combine static ggplot2 maps with interactive leaflet maps in a single dashboard
  • Publish an interactive HTML dashboard to RPubs
  • Write data-driven policy recommendations

Dashboard Requirements

Your dashboard must contain exactly 5 tabs:

Tab Content Key Visualization
National Context US county-level hardship choropleth with value boxes Static ggplot map + 2–3 value boxes
Local Patterns Block group hardship map linked to a detail table via crosstalk Leaflet map + DT table (linked)
Hot Spots LISA cluster map with hot/cold spot counts Static ggplot LISA map + value boxes + cluster table
Change Over Time Diverging change map (2015 vs. 2023) with improvement/worsening stats Static ggplot diverging map + value boxes
Policy Priority areas table + 3 written policy recommendations DT table + narrative text

Technical Requirements

  • At least 1 crosstalk-linked leaflet map + DT table
  • At least 2 value boxes with meaningful metrics
  • Published to RPubs (preferred), Quarto Pub, or GitHub Pages
  • Self-contained HTML under 25 MB
  • All code chunks labeled; echo: false in YAML header

Getting Started

  1. Download the Final_Project_Template.qmd file using the button above.
  2. Open it in RStudio.
  3. Update the title with your name.
  4. Follow the TODO comments in each section.
  5. Reuse code from your Labs 4, 5, and 6 solutions — most of the analytical code is the same.
  6. Render frequently to catch errors early.
  7. Publish to RPubs when you are satisfied with the result.

Publishing to RPubs

  1. Render your .qmd to HTML in RStudio (click Render).
  2. Create a free account at rpubs.com if you do not already have one.
  3. In the RStudio Viewer pane, click the Publish button (blue icon, top-right corner).
  4. Select RPubs as the destination and follow the prompts.
  5. Test the published URL in an incognito browser window.

Troubleshooting: If the file exceeds 25 MB, switch national maps from leaflet to static ggplot2. If crosstalk is not working on RPubs, ensure both the leaflet map and DT table reference the same SharedData object with a key argument (e.g., key = ~GEOID).
