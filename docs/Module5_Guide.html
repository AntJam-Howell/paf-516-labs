Temporal Change Analysis

Module Overview and Objectives

Modules 1-4 analyzed your study area at a single point in time. But neighborhoods are not static — they gentrify, decline, stabilize, or fluctuate over years and decades. This module adds the time dimension to your analysis. You will compare hardship index values across two American Community Survey (ACS) vintages to identify neighborhoods that are improving, deteriorating, or stable.

Temporal analysis with census data introduces unique methodological challenges. Census boundaries can change between years. Margins of error can make small changes statistically meaningless. Diverging color palettes must be chosen carefully to communicate the direction and magnitude of change. This module addresses all of these issues while building directly on the index and spatial analysis skills from previous labs.

After completing this module, you will be able to:

  • Retrieve and compare ACS data from two different vintages (e.g., 2011-2015 vs. 2019-2023) using tidycensus
  • Compute change scores for individual variables and composite indices
  • Address boundary changes between census vintages and understand when direct comparison is valid
  • Apply diverging color scales to effectively communicate positive and negative change
  • Assess whether observed changes exceed ACS margins of error
  • Evaluate the temporal stability of spatial patterns identified in Module 4
  • Connect neighborhood change to theories of gentrification, disinvestment, and neighborhood succession

Section 1: Multi-Year Census Comparison

The American Community Survey (ACS) provides rolling 5-year estimates. The tidycensus package lets you pull data from any available vintage by setting the year argument in get_acs(). For temporal comparison, you will retrieve the same variables for two different periods — for example, the 2011-2015 ACS and the 2019-2023 ACS.

Computing Change Scores

The simplest approach is a raw change score: the value at time 2 minus the value at time 1. For the hardship index:

change = hardship_2023 - hardship_2015
  • Positive change: Hardship increased (neighborhood deteriorated)
  • Negative change: Hardship decreased (neighborhood improved)
  • Near zero: Neighborhood was stable

You can also compute percentage change for individual variables, but be cautious with variables that have small denominators — percentage change can be unstable when the baseline is small.

Important: Index Comparability

When comparing z-score-based indices across time, you must decide: do you standardize each year's data using that year's mean and standard deviation, or using a pooled mean and standard deviation? If each year is standardized separately, a z-score of 1.0 in 2015 and 1.0 in 2023 both mean "one standard deviation above that year's average" — but the absolute level of hardship could be very different. For valid change scores, consider standardizing both years against a common baseline.

Section 2: Boundary Changes and Data Comparability

Census geographies are not permanent. The Census Bureau redraws tract and block group boundaries with each decennial census (2010, 2020). This means:

  • ACS 2011-2015 data uses 2010 census boundaries
  • ACS 2019-2023 data uses 2020 census boundaries

If boundaries changed in your study area, you cannot directly subtract one year's value from another for affected geographies — the areas are literally different shapes containing different populations.

Strategies for Handling Boundary Changes

  • Compare within the same boundary vintage: Use two ACS years that share the same boundaries (e.g., 2015 and 2019, both on 2010 boundaries). This is the simplest and most defensible approach
  • Crosswalk files: The Census Bureau and NHGIS provide relationship files that map old boundaries to new ones, allowing approximate comparison
  • Visual comparison only: Map both years side by side without computing change scores — readers can see the broad patterns without relying on unit-level subtraction

In the lab, you will use the approach that best fits your study area, but you must explicitly acknowledge the boundary issue in your analysis.

Section 3: Diverging Color Scales for Change Maps

Change data has a meaningful midpoint (zero = no change), which calls for a diverging color palette rather than a sequential one. Diverging palettes use two distinct hues radiating from a neutral center:

  • Blue-to-Red: Common convention where red signals "alarm" (increasing hardship) and blue signals improvement
  • Purple-to-Orange: A colorblind-friendly alternative
  • Brown-to-Teal: Another accessible option from ColorBrewer

Design Decisions

  • Center the midpoint on zero: The neutral color must correspond to "no change," not the median of your data. In ggplot2, use scale_fill_gradient2(midpoint = 0) or scale_fill_distiller(type = "div")
  • Symmetric limits: Set the minimum and maximum of the color scale to equal absolute values (e.g., -2 to +2) so that the visual weight of positive and negative changes is balanced
  • Handle outliers: Extreme values can compress the rest of the scale. Consider winsorizing (capping extreme values) or using a discrete classification

Section 4: Margins of Error and Statistical Significance of Change

ACS estimates come with margins of error (MOE), and these margins can be substantial for small geographies like block groups. A change from 15% to 18% poverty may look meaningful on a map, but if both estimates have MOEs of +/- 5 percentage points, the "change" is well within sampling noise.

Testing Whether Change Is Significant

The Census Bureau provides guidance for computing a z-test for the difference between two ACS estimates:

z = (estimate_2 - estimate_1) / sqrt(SE_1^2 + SE_2^2)

where SE = MOE / 1.645 for 90% confidence intervals. If |z| > 1.645, the change is statistically significant at the 90% level.

Practical implication: Your change map should distinguish between statistically significant changes and noise. Consider using hatching, transparency, or a separate significance layer to flag block groups where the observed change exceeds the margin of error.

Key Terms

TermDefinition
Change ScoreThe difference between a variable's value at two time points (time 2 minus time 1)
ACS VintageA specific 5-year period of American Community Survey data (e.g., 2019-2023)
Diverging Color ScaleA color palette with two hues radiating from a neutral midpoint, used for data with a meaningful center
Margin of Error (MOE)The range around an ACS estimate within which the true value likely falls (90% confidence)
Boundary ChangeRedrawing of census geographic boundaries between decennial censuses, complicating temporal comparison
Temporal StabilityThe degree to which spatial patterns persist over time
GentrificationNeighborhood change characterized by rising incomes, property values, and demographic shifts
WinsorizingCapping extreme values at a specified percentile to prevent outliers from dominating visualizations

Yellowdig Topic

Neighborhood change is often framed as either "improvement" or "decline," but whose perspective defines which is which? Consider:

  1. A block group's poverty rate dropped from 35% to 15% over 8 years. Is this good news? For whom? What might be happening that the numbers do not show?
  2. How would you distinguish gentrification (displacement of existing residents) from genuine neighborhood improvement (existing residents benefiting) using census data alone?
  3. Should policy target neighborhoods that are currently high-hardship, or neighborhoods where hardship is increasing fastest? What are the trade-offs?
